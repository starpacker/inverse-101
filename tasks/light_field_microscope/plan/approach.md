# Approach

## Problem Statement

Recover a 3D fluorescent volume from a single 2D light-field measurement using
the wave-optics model of Broxton et al. and Richardson-Lucy 3D deconvolution.

This cleaned task contains two related validations:

1. A primary benchmark with four thickness-1 volumes whose occupied slices are
   at `0`, `-30`, `-50`, and `-70` `um`.
2. A generalized five-slice demo with reconstruction depths
   `[-100, -75, -50, -25, 0]` `um` and a target located at `-50` `um`.

The first case mirrors the planar USAF target experiment from the paper. The
second case restores the intended interpretation of light-field microscopy as a
volume-reconstruction problem.

## Mathematical Formulation

### Forward Model

For an incoherent fluorescent specimen volume `v(x, y, z)`, the camera records

$$
m(s, t) \sim \mathrm{Poisson}\!\left(
\iiint h(s, t; x, y, z)\,v(x, y, z)\,dx\,dy\,dz
\right),
$$

where `h(s, t; x, y, z)` is the wave-optics intensity PSF for a point source at
voxel `(x, y, z)`.

The PSF is generated by the Broxton pipeline:

1. Compute the field from the objective to the MLA plane with a Debye model.
2. Multiply by the tiled quadratic microlens phase mask.
3. Propagate from the MLA to the sensor.
4. Square the magnitude to obtain an intensity kernel.

After discretization,

$$
\mathbf{y} \sim \mathrm{Poisson}(H\mathbf{v}),
$$

where `\mathbf{v}` stacks the unknown voxels and `H` is the depth-dependent PSF
bank.

### Thickness-1 Benchmark as a Special Case

The planar-target benchmark still uses the same volume formulation, but with

$$
v(x, y, z) = x(x, y)\,\delta(z-z_0).
$$

In other words, the unknown is a volume with exactly one occupied slice. The
main benchmark therefore evaluates the full 3D algorithm under the convenient
restriction that the depth axis has length `1`.

### Inverse Problem

Recover the non-negative volume by maximizing the Poisson likelihood:

$$
\hat{\mathbf{v}} =
\arg\min_{\mathbf{v} \ge 0}
\left[-\log p(\mathbf{y} \mid H\mathbf{v})\right].
$$

Using standard Richardson-Lucy updates,

$$
\mathbf{v}^{(q+1)} =
\mathbf{v}^{(q)} \odot
\frac{H^\top\!\left(\mathbf{y} \oslash (H\mathbf{v}^{(q)} + \varepsilon)\right)}
{H^\top \mathbf{1}},
$$

where `\odot` and `\oslash` are elementwise multiplication and division.

## Solution Strategy

### Step 1: Load Metadata and Reference Settings

Read `data/meta_data.json` and extract:

- microscope and MLA parameters
- benchmark occupied depths `[0, -30, -50, -70]` `um`
- generalized volume-demo depth grid `[-100, -75, -50, -25, 0]` `um`
- synthetic noise level (`poissonScale`)

Note: solver parameters (RL iteration count, theta samples, kernel tolerance)
are hardcoded as named constants in `main.py` and are not stored in
`meta_data.json`.

### Step 2: Build Wave-Optics Systems on the Required Depth Grids

Use `src/physics_model.py` to write the YAML config and build the Broxton
wave-optics system directly:

- one singleton-depth system per benchmark case
- one five-slice system for the generalized volume demo

### Step 3: Generate Synthetic Volumes and Observations

Construct a square-windowed line-pair target and embed it into the requested
volume:

- benchmark: one occupied slice per case
- volume demo: one occupied slice at `-50` `um` within the five-slice grid

Forward-project each volume to a 2D light-field image and add Poisson noise.

### Step 4: Reconstruct the Volume with Richardson-Lucy

Run Richardson-Lucy deconvolution via `solvers.run_richardson_lucy()`, which
implements the full RL multiplicative update directly using the system's
`forward_project` and `backward_project` methods.

- benchmark outputs are singleton-depth volumes; their only slice is compared
  against the planar target and used for boundary metrics
- the generalized demo keeps the full five-slice reconstruction and reports its
  slice-energy distribution over depth

### Step 5: Compare Against a Conventional Microscope Baseline

For the benchmark cases, convolve the target with the defocused conventional
microscope PSF at the same source depth. Report MSE, dynamic-range NRMSE, SSIM,
and PSNR for both the conventional image and the wave-model reconstruction.

### Step 6: Save Standardized Artifacts

Write:

- `data/raw_data.npz` with light-field observations and benchmark depth grids
- `data/ground_truth.npz` with planar slices and thickness-1 ground-truth
  volumes
- `evaluation/reference_outputs/cases.npz` with benchmark observations,
  volumes, baselines, reconstructions, and profiles
- `evaluation/reference_outputs/volume_demo.npz` with the five-slice
  generalized volume demo
- `evaluation/reference_outputs/*.png` and `*.json` for notebook review
- `output/reconstruction.npy` for end-to-end scoring

## Expected Results

At the native plane (`0 um`), the conventional microscope can be competitive
because little axial discrimination is needed. Away from the native plane, the
wave-model reconstruction should remain closer to the target than the
conventional baseline because the MLA provides depth-dependent angular diversity
that the 3D deconvolution can exploit.

The generalized five-slice demo should show a stronger result: although the
sensor still records only one 2D light-field image, the reconstructed axial
energy should peak at the true target depth `-50` `um`, while slices at
`-100`, `-75`, `-25`, and `0` `um` remain much weaker. That is the intended
volume-reconstruction behavior described in the Broxton paper.
