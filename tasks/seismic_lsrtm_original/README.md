# Least-Squares Reverse-Time Migration (From-Scratch C-PML Implementation)

> Recover a 2D subsurface reflectivity image from synthetic surface seismograms using a from-scratch acoustic Born LSRTM solver, where the inversion is difficult because weak band-limited scattered arrivals must be isolated from strong direct waves.

> Domain: Earth Science | Keywords: wave imaging, inverse scattering, seismic migration | Difficulty: Medium

## Background
Least-squares reverse-time migration (LSRTM) estimates a reflectivity-like scattering image from seismic shot gathers. A smooth migration velocity model is treated as known, and the unknown image explains the scattered portion of the measured pressure field.

For synthetic seismic benchmarks, the observed data are generated by propagating acoustic waves through a true velocity model $v_{true}(y, x)$. The inversion does not recover $v_{true}$ directly. Instead, it reconstructs a scattering potential $s(y, x)$ that perturbs wave propagation around the known smooth background $v_{mig}(y, x)$.

## Problem Description
Let $d_{obs}$ denote the observed shot gathers, $d_{0} = F(v_{mig})$ the direct-arrival data predicted by the smooth background model, and $d_{scat} = d_{obs} - d_{0}$ the scattered data. Under the acoustic Born approximation,

$$d_\mathrm{scat} \approx L(v_\mathrm{mig})s,$$

where $L(v_{mig})$ is the linearized forward operator around the migration velocity and $s(y, x)$ is the unknown scattering image. The inverse problem is

$$\hat{s} = \arg\min_s \frac{1}{2}\lVert L(v_\mathrm{mig})s - d_\mathrm{scat} \rVert_2^2.$$

The measurements are limited in bandwidth, recording time, and acquisition aperture. Those limitations remove high-wavenumber information and make the inversion ill-conditioned even when the background velocity is known.

## Data Description
All arrays stored in `.npz` files follow the batch-first convention `(1, ...)`.

### `data/raw_data.npz`

| Key | Shape | Dtype | Description |
|-----|-------|-------|-------------|
| `v_mig` | `(1, 600, 250)` | float32 | Smooth migration velocity model used by the direct-arrival and Born propagators (m/s) |
| `observed_data` | `(1, 20, 100, 300)` | float32 | Observed pressure shot gathers for 20 shots, 100 receivers per shot, and 300 time samples (pressure amplitude units) |

### `data/ground_truth.npz`

| Key | Shape | Dtype | Description |
|-----|-------|-------|-------------|
| `v_true` | `(1, 600, 250)` | float32 | True Marmousi P-wave velocity model used to synthesize the observations (m/s) |

### `data/meta_data.json`
`meta_data.json` stores imaging parameters only: full-model and cropped-model sizes, spatial grid spacing, acquisition geometry, source wavelet settings, and time sampling. It intentionally excludes solver hyperparameters such as optimizer type, iteration count, and loss scaling.

## Method Hints
Use a linearized acoustic LSRTM workflow. First predict the direct arrivals in the smooth migration model and subtract them from the observations. Then solve the least-squares Born inverse problem for the scattering image using the same acquisition geometry and wave propagation model in the forward and adjoint computations.

This task uses a from-scratch C-PML finite-difference propagator rather than Deepwave. A suitable implementation should preserve the original modeling logic: explicit wave stepping, surface source injection, receiver sampling, and absorbing boundaries for both the background and scattered wavefields.
